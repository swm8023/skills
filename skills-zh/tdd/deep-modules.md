# 深模块（Deep Modules）

摘自《A Philosophy of Software Design》：

**深模块（Deep module）** = 小接口 + 大量实现

```
┌─────────────────────┐
│   Small Interface   │  ← 方法少，参数简单
├─────────────────────┤
│                     │
│                     │
│  Deep Implementation│  ← 复杂逻辑被隐藏
│                     │
│                     │
└─────────────────────┘
```

**浅模块（Shallow module）** = 大接口 + 少量实现（应避免）

```
┌─────────────────────────────────┐
│       Large Interface           │  ← 方法多，参数复杂
├─────────────────────────────────┤
│  Thin Implementation            │  ← 仅仅是穿透转发
└─────────────────────────────────┘
```

设计接口时，问自己：

- 能不能减少方法的数量？
- 能不能简化参数？
- 能不能在内部隐藏更多复杂度？
